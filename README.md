# OpenMTOps — F&O Options Trading Assistant

> Self-hosted Flask + SocketIO web app for Indian F&O trading. Reads signals from Telegram, executes into Dhan brokerage, and provides live option chain, OI tracker, and strategy dashboard.

---

## Quick Start

```bash
git clone <repo> openmtops
cd openmtops
python -m venv venv
source venv/bin/activate          # Windows: venv\Scripts\activate
pip install -r requirements.txt
python app.py                     # opens on http://127.0.0.1:5000
```

Open the browser — if no credentials are configured, the **setup wizard** starts automatically. It walks you through Telegram API credentials, Telegram authentication, Dhan broker credentials, and optional settings (PIN + channel ID). At the end it writes everything to `.env` so credentials survive restarts.

### Power User (.env path)

If you already have a `.env` file with all required credentials, the wizard is skipped on startup:

```bash
cp .env.example .env
# fill in TELEGRAM_API_APP, TELEGRAM_API_HASH, DHAN_CLIENTID, DHAN_ACCESSTOKEN
python app.py   # wizard skipped — goes straight to the app
```

### Test mode (no credentials)

```bash
TESTING=1 python app.py
# or: make test
```

Injects mock modules for broker, price feed, and candle service. The setup wizard and auth guard are bypassed. All pages load with dummy data.

## What It Does

1. Reads trading signals from a private **Telegram channel** using Telethon
2. Shows parsed tip cards (symbol, strike, CE/PE, entry, SL, targets)
3. Lets the user click-to-execute into **Dhan** brokerage via dhanhq SDK
4. Watches price live over **DhanHQ WebSocket** and auto-exits at SL/targets
5. Provides an **Option Analyzer** (option chain, OI, Greeks, live LTP)
6. Provides an **OI Tracker** (records OI buildup per strike over time)
7. Provides a **Strategy Dashboard** (regime, phase, EMA, OI walls — powered by stored 5-min candles)
8. Sends **in-app notifications** (bell dropdown, real-time via SocketIO) for new tips and ENTER signals

## Credential Persistence

Credentials are stored in two places that are always kept in sync:

```
config.json          (runtime — highest priority, written by wizard & Settings page)
    ↓ synced to
.env                 (durable — loaded by load_dotenv() on restart)
```

The Settings page (`/settings`) updates `config.json` immediately (no restart needed) and also rewrites `.env` so credentials persist across restarts.

## Environment Variables (`.env`)

| Variable | Required | Description |
|---|---|---|
| `SECRET_KEY` | Yes | Flask session signing key — auto-generated by wizard if absent |
| `APP_PIN` | Optional | 4–8 digit PIN gate. Leave blank to disable. |
| `TESTING` | No | Set to `1` for dummy data mode |
| `DHAN_CLIENTID` | Yes | Dhan Client ID |
| `DHAN_ACCESSTOKEN` | Yes | Dhan JWT access token (expires annually) |
| `DHAN_APIKEY` | Optional | Dhan API key (future use) |
| `DHAN_APISECRET` | Optional | Dhan API secret (future use) |
| `TELEGRAM_API_APP` | Yes | Telegram API App ID (from my.telegram.org/apps) |
| `TELEGRAM_API_HASH` | Yes | Telegram API Hash (from my.telegram.org/apps) |
| `TELEGRAM_CHANNEL_ID` | Optional | Numeric ID of the channel to read tips from |
| `TELETHON_SESSION` | Optional | Override path for `anon.session` file (useful for Docker) |

## Production Hosting

See [deployment/INSTALL.md](deployment/INSTALL.md) for step-by-step instructions.

Quick overview:
1. Clone to your server, create a virtualenv, `pip install -r requirements.txt`
2. Edit `deployment/openmtops.service` — replace paths with your install directory
3. Install and start the systemd service
4. Configure nginx as a reverse proxy (handles WebSocket upgrade)
5. Open the app URL — the setup wizard runs automatically on first visit

## Key Architecture Notes

**Single-user design**: `_trade`, `_chain`, `_tracker` are module-level dicts — not per-session. This is intentional for a personal trading tool. The PIN gate (`APP_PIN`) protects the app if hosted on a public IP.

**Market hours**: Candle service and signal notifier only run Mon–Fri 09:15–15:30 IST. This is intentional for Indian F&O.

**Instrument IDs**: `NIFTY=13`, `BANKNIFTY=25`, etc. are Dhan platform constants — the same for all Dhan users.

**Telethon session**: `anon.session` must exist for Telegram to work. The setup wizard creates it. After that it persists across restarts. Override the path with `TELETHON_SESSION` (useful for Docker volumes).
